# ACL (Access Control List)

Компонента поддерживает управление списками контроля доступа (ACL) Kafka через Admin API librdkafka.

## Оглавление

| Метод | Описание |
|-------|----------|
| [ДобавитьACL](#добавитьacl) | Создание одной ACL-записи |
| [ОписатьACL](#описатьacl) | Поиск ACL-записей по фильтру |
| [УдалитьACL](#удалитьacl) | Удаление ACL-записей по фильтру |

## Справочные значения параметров

### Тип ресурса (resource_type)

| Значение | Описание |
|----------|----------|
| TOPIC | Топик |
| GROUP | Consumer group |
| BROKER | Брокер / кластер |
| TRANSACTIONAL_ID | Transactional id (продюсер) |
| ANY | Любой тип (только для фильтров) |

### Тип шаблона (pattern_type)

| Значение | Описание |
|----------|----------|
| LITERAL | Точное совпадение имени ресурса |
| PREFIXED | Совпадение по префиксу имени |
| MATCH | Расширенное совпадение (только для фильтров) |
| ANY | Любой тип шаблона (только для фильтров) |

### Операция (operation)

| Значение | Описание |
|----------|----------|
| ALL | Все операции |
| READ | Чтение |
| WRITE | Запись |
| CREATE | Создание |
| DELETE | Удаление |
| ALTER | Изменение |
| DESCRIBE | Получение описания |
| CLUSTER_ACTION | Действия над кластером |
| DESCRIBE_CONFIGS | Получение конфигурации |
| ALTER_CONFIGS | Изменение конфигурации |
| IDEMPOTENT_WRITE | Идемпотентная запись |
| ANY | Любая операция (только для фильтров) |

### Тип разрешения (permission_type)

| Значение | Описание |
|----------|----------|
| ALLOW | Разрешить |
| DENY | Запретить |
| ANY | Любой тип (только для фильтров) |

## Методы компоненты и их описание

### ДобавитьACL

```1c
ДобавитьACL(Брокер, ТипРесурса, ИмяРесурса, ТипШаблона, Принципал, Хост, Операция, ТипРазрешения, Таймаут = 10000)
```

**Брокер** - Строка. Адрес брокера.

**ТипРесурса** - Строка. Один из: TOPIC, GROUP, BROKER, TRANSACTIONAL_ID.

**ИмяРесурса** - Строка. Имя ресурса (например, имя топика или consumer group).

**ТипШаблона** - Строка. Один из: LITERAL, PREFIXED.

**Принципал** - Строка. Идентификатор субъекта в формате `User:имя` или `User:*` для всех.

**Хост** - Строка. Хост или IP клиента. Используйте `*` для любого хоста.

**Операция** - Строка. Одно из значений операции (см. выше).

**ТипРазрешения** - Строка. ALLOW или DENY.

**Таймаут** - Число. Таймаут ожидания ответа в миллисекундах. По умолчанию 10000.

Метод возвращает `Истина` при успешном создании ACL, `Ложь` - при ошибке.

Пример использования:

```1c
Компонента = Новый("AddIn.SimpleKafka1C.SimpleKafka1C");

// Разрешить пользователю alice читать топик orders с любого хоста
Если Компонента.ДобавитьACL("localhost:9092", "TOPIC", "orders", "LITERAL",
                             "User:alice", "*", "READ", "ALLOW") Тогда
    Сообщить("ACL создана");
Иначе
    Сообщить("Ошибка: " + Компонента.ПолучитьСообщениеОбОшибке());
КонецЕсли;

// Разрешить роли writers записывать во все топики с префиксом logs-
Компонента.ДобавитьACL("localhost:9092", "TOPIC", "logs-", "PREFIXED",
                       "User:writers", "*", "WRITE", "ALLOW");
```

### ОписатьACL

```1c
ОписатьACL(Брокер, ТипРесурса = "ANY", ИмяРесурса = "", ТипШаблона = "ANY",
          Принципал = "", Хост = "", Операция = "ANY", ТипРазрешения = "ANY", Таймаут = 10000)
```

**Брокер** - Строка. Адрес брокера.

**ТипРесурса** - Строка. Фильтр по типу ресурса. Значение `ANY` или пустая строка - любой тип.

**ИмяРесурса** - Строка. Фильтр по имени ресурса. Пустая строка - любое имя.

**ТипШаблона** - Строка. Фильтр по типу шаблона. `ANY`, `MATCH`, `LITERAL` или `PREFIXED`.

**Принципал** - Строка. Фильтр по principal. Пустая строка - любой principal.

**Хост** - Строка. Фильтр по хосту. Пустая строка - любой хост.

**Операция** - Строка. Фильтр по операции. `ANY` - любая.

**ТипРазрешения** - Строка. Фильтр по типу разрешения. `ANY` - любое.

**Таймаут** - Число. Таймаут ожидания ответа в миллисекундах.

Метод возвращает строку JSON со списком найденных ACL-записей.

Структура:
+ count - Количество найденных ACL
+ acls - Массив ACL-записей
  + resource_type - Тип ресурса
  + resource_name - Имя ресурса
  + pattern_type - Тип шаблона
  + principal - Principal (субъект)
  + host - Хост
  + operation - Операция
  + permission_type - Тип разрешения

Пример использования:

```1c
Компонента = Новый("AddIn.SimpleKafka1C.SimpleKafka1C");

// Получить все ACL кластера
РезультатJSON = Компонента.ОписатьACL("localhost:9092");
Сообщить(РезультатJSON);

// Найти все ACL для топика orders
РезультатJSON = Компонента.ОписатьACL("localhost:9092", "TOPIC", "orders", "LITERAL");
Сообщить(РезультатJSON);

// Найти все ACL пользователя alice
РезультатJSON = Компонента.ОписатьACL("localhost:9092", "ANY", "", "ANY",
                                       "User:alice", "", "ANY", "ANY");
Сообщить(РезультатJSON);
```

Пример ответа:

```json
{
    "count": "1",
    "acls": [
        {
            "resource_type": "Topic",
            "resource_name": "orders",
            "pattern_type": "Literal",
            "principal": "User:alice",
            "host": "*",
            "operation": "Read",
            "permission_type": "Allow"
        }
    ]
}
```

### УдалитьACL

```1c
УдалитьACL(Брокер, ТипРесурса = "ANY", ИмяРесурса = "", ТипШаблона = "ANY",
          Принципал = "", Хост = "", Операция = "ANY", ТипРазрешения = "ANY", Таймаут = 10000)
```

Параметры аналогичны методу `ОписатьACL` - это фильтр для выбора удаляемых записей.

> **Внимание!** При использовании очень широких фильтров (все пустые) будут удалены **все** ACL в кластере. Используйте с осторожностью.

Метод возвращает строку JSON со списком удалённых ACL.

Структура:
+ deleted_count - Количество удалённых ACL
+ deleted_acls - Массив удалённых записей (поля аналогичны ОписатьACL)

Пример использования:

```1c
Компонента = Новый("AddIn.SimpleKafka1C.SimpleKafka1C");

// Удалить все ACL для топика orders
РезультатJSON = Компонента.УдалитьACL("localhost:9092", "TOPIC", "orders", "LITERAL");
Если ПустаяСтрока(РезультатJSON) Тогда
    Сообщить("Ошибка: " + Компонента.ПолучитьСообщениеОбОшибке());
Иначе
    Сообщить(РезультатJSON);
КонецЕсли;

// Удалить конкретную ACL (точное совпадение всех полей)
Компонента.УдалитьACL("localhost:9092", "TOPIC", "orders", "LITERAL",
                      "User:alice", "*", "READ", "ALLOW");
```

Пример ответа:

```json
{
    "deleted_count": "1",
    "deleted_acls": [
        {
            "resource_type": "Topic",
            "resource_name": "orders",
            "pattern_type": "Literal",
            "principal": "User:alice",
            "host": "*",
            "operation": "Read",
            "permission_type": "Allow"
        }
    ]
}
```

## Требования

Для работы с ACL кластер Kafka должен быть настроен с authorizer - например, `kafka.security.authorizer.AclAuthorizer`. Пользователь, под которым подключается компонента, должен иметь административные привилегии для управления ACL.

Детали см. в [документации Kafka по авторизации](https://kafka.apache.org/documentation/#security_authz).
